# Update a Memory Map

## Goal

Revise an existing memory map without losing useful structure.

## Rules

- Preserve good nodes and links when still relevant
- Keep node ids stable when practical
- Add missing novelty, analogy, or association only where they improve recall
- Remove redundant nodes if they add clutter but no memory value

## Typical Reasons to Update

- the source note changed
- the user learned a better analogy
- more vault context is now available
- the map feels too dense or too shallow
